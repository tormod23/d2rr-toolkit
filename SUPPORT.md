# Support

Thanks for using D2RR Toolkit. This file explains how to get help.

## Before you open an issue

Skim these first - most questions are answered directly:

- [`README.md`](README.md) - install, quickstart, environment variables.
- [`docs/CLI_REFERENCE.md`](docs/CLI_REFERENCE.md) - every command and option.
- [`docs/ARCHITECTURE.md`](docs/ARCHITECTURE.md) - package layout and data flow.
- [`docs/spec/d2s_format_spec.md`](docs/spec/d2s_format_spec.md) - binary save-file reference.

## Usage questions

GitHub Discussions is not currently enabled on this repository. Open an
issue with the `question` label. The maintainer will re-triage if the
answer belongs in the docs instead.

## Bug reports and feature requests

Use the bug-report or feature-request template on the [issue tracker](https://github.com/tormod23/d2rr-toolkit/issues/new/choose).
See [`CONTRIBUTING.md`](CONTRIBUTING.md) for the PR conventions you will
need when submitting a fix.

## Security issues

Do **not** file public issues for security problems. See
[`SECURITY.md`](SECURITY.md) for the private-advisory path and the SLA.

---

This is a pre-1.0 project with one maintainer. Responses are best-effort;
security issues follow the SLA in `SECURITY.md`.
